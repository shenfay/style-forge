# 组件命名规范

基于 Ant Design 和 Material UI 的命名约定，采用国际主流UI框架标准命名。

## 布局类 (Layout)

| 组件名 | 说明 | 使用场景 |
|--------|------|---------|
| `Container` | 容器 | 页面主容器，控制最大宽度和边距 |
| `Grid` | 宫格布局 | 分类导航、图标网格 |
| `Section` | 区块 | 页面内容区块（包含标题+内容） |
| `Stack` | 堆叠布局 | 垂直/水平排列的子元素 |

## 数据展示类 (Data Display)

| 组件名 | 说明 | 使用场景 |
|--------|------|---------|
| `Card` | 卡片 | 商品卡片、内容卡片 |
| `List` | 列表 | 商品列表、订单列表 |
| `Image` | 图片 | 商品图、Banner图 |
| `Avatar` | 头像 | 用户头像、品牌Logo |
| `Badge` | 徽章 | 角标、状态标记 |
| `Tag` | 标签 | 分类标签、状态标签 |
| `Typography` | 排版 | 标题、正文、辅助文字 |

## 导航类 (Navigation)

| 组件名 | 说明 | 使用场景 |
|--------|------|---------|
| `NavBar` | 顶部导航栏 | 页面顶部导航 |
| `TabBar` | 底部导航栏 | 移动端底部Tab切换 |
| `Tabs` | 选项卡 | 内容分组切换 |
| `Segmented` | 分段控制器 | 视图切换、筛选器 |
| `Breadcrumb` | 面包屑 | 页面层级导航 |

## 输入类 (Input)

| 组件名 | 说明 | 使用场景 |
|--------|------|---------|
| `Button` | 按钮 | 操作按钮 |
| `SearchBar` | 搜索栏 | 顶部搜索框 |
| `Input` | 输入框 | 文本输入 |
| `Switch` | 开关 | 布尔值切换 |
| `Radio` | 单选框 | 单选选项 |
| `Checkbox` | 复选框 | 多选选项 |

## 反馈类 (Feedback)

| 组件名 | 说明 | 使用场景 |
|--------|------|---------|
| `Carousel` | 轮播图 | Banner轮播、图片轮播 |
| `Modal` | 弹窗 | 对话框、确认框 |
| `Toast` | 轻提示 | 操作反馈提示 |

## 特殊组件

| 组件名 | 说明 | 使用场景 |
|--------|------|---------|
| `SectionHeader` | 区块标题 | "品牌专区"、"新品推荐"等区块标题 |
| `PriceTag` | 价格标签 | 商品价格展示（原价+现价） |
| `PromoCard` | 营销卡片 | 限时抢购、新人专享等营销模块 |

## 命名原则

1. **使用名词** - 组件名应该是名词，描述"是什么"而非"做什么"
2. **首字母大写** - PascalCase 命名法
3. **避免缩写** - 使用完整单词（Button 而非 Btn）
4. **语义清晰** - 名称应直观表达组件用途
5. **符合惯例** - 优先采用 Ant Design / Material UI 的标准命名

## 配置关联

所有组件样式通过**全局设计令牌（Design Tokens）**控制：

```
StyleConfig (配置系统)
    ↓
Design Tokens (设计令牌映射)
    ↓
Components (组件样式应用)
```

示例：
- `config.cornerRadius = 'large'` → 所有 Card、Button 圆角 = 12px
- `config.primaryColor = '#2E7D32'` → Button 主色、Tag 背景色
- `config.cardStyle = 'shadow'` → 所有 Card 添加阴影效果

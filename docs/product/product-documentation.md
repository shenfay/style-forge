# Style Forge - 场景化设计配置器 产品文档

## 产品定位

**场景化设计配置器** — 不只是选颜色，而是在真实 UI 组件中实时预览效果，一键导出可用配置。

### 与现有配色网站的核心差异

| 现有配色网站 | Style Forge |
|-------------|-------------|
| 只提供色板组合 | 真实 UI 组件预览 |
| 抽象色块展示 | 手机/PC 模拟器效果 |
| 无导出功能 | Tailwind Config + AI 提示词 |
| 单一维度 | 20 维配置矩阵 |

### 目标用户

1. **前端开发者** — 快速生成项目设计系统
2. **UI 设计师** — 探索配色方案，验证可访问性
3. **独立开发者** — 一键导出 AI 提示词给大模型生成代码
4. **产品经理** — 快速原型验证

## 设计系统规范

### 20 维配置矩阵

#### 色彩系统（10 维）

| 维度 | 说明 | 影响范围 |
|------|------|---------|
| **主色** | 品牌核心色 | 按钮、链接、高亮 |
| **辅助色** | 次要交互色 | 次要按钮、标签 |
| **强调色** | 点缀色 | 图标、装饰元素 |
| **页面背景色** | 全局底色 | 页面整体背景 |
| **卡片背景色** | 容器底色 | 卡片、面板 |
| **标题色** | 标题文字色 | 各层级标题 |
| **正文色** | 主要文字色 | 段落内容 |
| **辅助色** | 次要文字色 | 提示文字、时间戳 |
| **成功色** | 正向反馈色 | 成功提示、安全标签 |
| **警告色** | 警示色 | 警告提示、错误提示 |

#### 形状系统（6 维）

| 维度 | 选项 | 说明 | 影响范围 |
|------|------|------|---------|
| **圆角大小** | small / medium / large | 风格基调 | 按钮、卡片、标签 |
| **卡片样式** | border / shadow / borderless | 容器视觉表现 | 内容区块 |
| **按钮样式** | gradient / solid / wireframe | 行动召唤力度 | 操作按钮 |
| **标签样式** | rounded / text-only | 信息密度 | 标签组件 |
| **标题栏样式** | white-underline / frosted-glass / colored-bg | 空间感和层级 | 导航区 |
| **切换器样式** | underline / pill / capsule | 交互亲密度 | 选项卡组件 |

#### 间距系统（4 维）

| 维度 | 选项 | 说明 | 影响范围 |
|------|------|------|---------|
| **内边距** | compact / medium / relaxed | 容器内部间距 | 卡片、面板 |
| **卡片间距** | small / medium / large | 卡片之间间距 | 列表、网格 |
| **区块间距** | small / medium / large | 大区块间距 | 页面分区 |
| **元素间距** | compact / medium / relaxed | 小元素间距 | 按钮组、标签组 |

#### 文字排版（5 维）

| 维度 | 选项 | 说明 | 影响范围 |
|------|------|------|---------|
| **标题装饰** | left-accent / right-accent / bottom-accent / plain | 标题视觉引导 | 区块标题 |
| **标题大小** | small / medium / large | 标题字号 | 各层级标题 |
| **标题字重** | normal / medium / bold | 标题粗细 | 标题文字 |
| **正文字号** | small / medium / large | 正文字号 | 段落内容 |
| **行高** | compact / medium / relaxed | 文字行间距 | 所有文本 |

### 配置联动机制

已实现的联动规则：
- 大圆角 + 强阴影 → 增强视觉层次
- 大圆角 + 大内边距 → 更饱满的视觉感受
- 大标题 + 粗装饰线 → 强化视觉引导
- 宽松行高 + 大段落间距 → 提升可读性

| **食品健康** | `food` | 扫码结果、成分分析 | 食品、健康、餐饮 |

## 技术架构

### 零后端架构

| 特性 | 实现方式 |
|------|----------|
| 配置存储 | URL 参数 + Base64 编码 |
| 模板数据 | 静态 JSON 文件 |
| 状态管理 | URL config 驱动 + Zustand |
| 分享功能 | 复制完整 URL（含 config） |

### 技术栈

- **框架**：React 18 + TypeScript
- **构建**：Vite 5
- **样式**：Tailwind CSS 4 + 设计令牌系统
- **状态**：Zustand

| 平台 | 说明 |
|------|------|
| **Vercel** | 主推，免费，CI/CD |

## LLM 集成方案

### 核心交互流程

```
用户 ──▶ LLM ──▶ Style Forge ──▶ 用户 ──▶ LLM ──▶ 代码生成
          │                      │                      │
          │ "帮我做一个电商      │ "打开链接，          │ "这是确认的
          │  商品详情页"          │  调整配置"          │  设计方案"
          ▼                      ▼                      ▼
     生成预览URL            预览/调整配置            粘贴提示词
```

### URL 参数规范

```
https://style.atmedia.fun/preview?scene={scene}&device={device}&template={template}&config={base64}
```

| 参数 | 必填 | 说明 | 可选值 |
|------|------|------|--------|
| `scene` | 是 | 场景类型 | `ecommerce`, `content`, `food`, `saas` 等 |
| `device` | 是 | 设备类型 | `mobile`, `desktop` |
| `template` | 是 | 页面模板 | `home`, `detail`, `list`, `profile` 等 |
| `config` | 否 | Base64 编码的完整样式配置 | 从预设或用户调整派生 |

## 附录

### 风险色定义（固定）

以下颜色不参与配置，作为语义色固定使用：

```typescript
const riskColors = {
  danger:  { bg: '#FEF2F2', text: '#DC2626', border: '#FECACA' },
  warning: { bg: '#FFFBEB', text: '#D97706', border: '#FDE68A' },
  safe:    { bg: '#F0FDF4', text: '#16A34A', border: '#BBF7D0' }
}
```

### Schema 规范

完整的数据模型定义包含在 `style-schema.json` 文件中，涵盖配置、组件库和模板的标准化结构。

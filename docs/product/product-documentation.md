# Style Forge - 场景化设计配置器 产品文档

## 产品概述

Style Forge 是一款面向开发者、设计师和产品经理的**场景化设计配置器**。它不同于传统的配色网站，而是通过在真实UI组件中实时预览效果，帮助用户快速生成并导出可用的设计配置。

### 核心价值主张

- **不只是选颜色**：在真实UI组件中实时预览效果
- **一键导出可用配置**：支持 Tailwind Config 和 AI 提示词导出
- **8维配置矩阵**：提供7,776种风格方案组合
- **多场景模板**：覆盖食品健康、电商零售、SaaS工具等多个领域

### 目标用户

1. **前端开发者** - 快速生成项目设计系统
2. **UI设计师** - 探索配色方案，验证可访问性
3. **独立开发者** - 一键导出AI提示词给大模型生成代码
4. **产品经理** - 快速原型验证

## 设计系统规范

### 8维配置矩阵

| 维度 | 选项 | 说明 | 影响范围 |
|------|------|------|---------|
| **页面底色** | 纯白/#FFFFFF, 暖沙/#F9F8F4, 淡黄/#FFFDF5, 淡绿/#F5FAF5 | 决定整体视觉温度 | 全局背景 |
| **主题色** | 森林绿/#2E7D32, 草绿/#5F7D2E, 青绿/#2E7D67, 纯黑/#000000 | 品牌识别与交互焦点 | 按钮、标题、高亮 |
| **圆角大小** | 小/8px, 中/16px, 大/24px | 风格基调 | 按钮、卡片、标签 |
| **卡片样式** | 边框/shadow, 阴影/纯阴影, 无装饰/borderless | 容器视觉表现 | 内容区块 |
| **标题栏** | 纯白+下划线, 毛玻璃/frosted, 彩色底色 | 空间感和层级 | 导航区 |
| **切换器** | 下划线/underline, 药丸/pill, 胶囊/capsule | 交互亲密度 | 选项卡组件 |
| **按钮** | 渐变/gradient, 纯色/solid, 线框/wireframe | 行动召唤力度 | 操作按钮 |
| **标签** | 圆角底色/rounded, 纯文字/text-only | 信息密度 | 徽章组件 |

### 配置组合数

理论组合: `4×4×3×3×3×3×3×2 = 7,776` 种风格方案

## 技术架构

### 零后端架构

| 特性 | 实现方式 |
|------|----------|
| 配置存储 | URL 参数 + Base64 编码 |
| 模板数据 | 静态 JSON 文件 |
| 状态管理 | React Hooks + URL |
| 分享功能 | 复制 URL |

### 技术栈

- **框架**：React 18 + TypeScript
- **构建**：Vite 5
- **样式**：Tailwind CSS 3
- **组件**：函数组件 + Hooks

### 部署方案

| 平台 | 说明 |
|------|------|
| **Vercel** | 主推，免费，CI/CD |
| **Cloudflare Pages** | 备选，全球 CDN |
| **Netlify** | 备选，静态托管 |

#### 域名
- 主域名：`styleforge.dev`
- 预览路径：`/preview?scene=...`

## LLM 集成方案

### 核心交互流程

```
用户 ──▶ LLM ──▶ Style Forge ──▶ 用户 ──▶ LLM ──▶ 代码生成
          │                      │                      │
          │ "帮我做一个电商      │ "打开链接，          │ "这是确认的
          │  商品详情页"          │  调整配置"          │  设计方案"
          ▼                      ▼                      ▼
     生成预览URL            预览/调整配置            粘贴提示词
```

### URL 参数规范

```
https://style-forge.dev/preview?scene={scene}&device={device}&template={template}
```

| 参数 | 必填 | 说明 | 可选值 |
|------|------|------|--------|
| `scene` | 是 | 场景类型 | `food`, `ecommerce`, `saas`, `media`, `social`, `finance` |
| `device` | 是 | 设备类型 | `mobile`, `desktop` |
| `template` | 是 | 页面模板 | `home`, `detail`, `list`, `form`, `settings`, `result` |
| `config` | 否 | Base64 编码的样式配置 | 从预设配置派生 |

### 交互方式演进

- **v1.0（当前阶段）**：URL + 提示词
- **v1.5**：轻量 API（Edge Functions）
- **v2.0**：MCP 集成

## 附录

### 风险色定义（固定）

这些颜色不参与配置，作为语义色固定使用：

```typescript
const riskColors = {
  danger: { bg: '#FEF2F2', text: '#DC2626', border: '#FECACA' },
  warning: { bg: '#FFFBEB', text: '#D97706', border: '#FDE68A' },
  safe: { bg: '#F0FDF4', text: '#16A34A', border: '#BBF7D0' }
}
```

### Schema 规范

完整的数据模型定义包含在 `style-schema.json` 文件中，涵盖配置、组件库和模板的标准化结构。
